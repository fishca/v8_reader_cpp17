## 🎯 Этапы переноса

### Этап 1: Завершение базовой инфраструктуры (Высокий приоритет)
**Срок:** 1-2 недели  
**Ответственный:** Core Team

#### 1.1. Завершение работы с типами данных
**Файлы:** `src/core/include/v8reader/core/Types.h`, `src/core/src/Types.cpp`

**Задачи:**
- [ ] Реализовать класс `V8Types` с методами для работы с типами данных 1С
- [ ] Добавить поддержку составных типов данных
- [ ] Реализовать конвертацию между `UnicodeString` → `QString`
- [ ] Создать утилиты для работы с `QByteArray` вместо `TMemoryStream`
- [ ] Реализовать замену `TGUID` → `QUuid` с методами парсинга

**Критерии готовности:**
- ✅ Все типы данных 1С корректно определяются
- ✅ Конвертация строк работает для UTF-16LE ↔ UTF-8
- ✅ GUID читаются из бинарных данных

#### 1.2. Доработка парсера дерева свойств
**Файлы:** `src/core/include/v8reader/core/TreeReader.h`, `src/core/src/TreeReader.cpp`

**Задачи:**
- [ ] Покрыть юнит-тестами все методы парсинга
- [ ] Добавить обработку граничных случаев (пустые значения, экранирование)
- [ ] Оптимизировать производительность для больших файлов
- [ ] Добавить логирование ошибок парсинга

**Критерии готовности:**
- ✅ 95%+ покрытие тестами
- ✅ Корректный парсинг всех форматов дерева свойств
- ✅ Обработка ошибок без падений

#### 1.3. Реализация V8File и V8Catalog
**Файлы:** `V8File.h/.cpp`, `V8Catalog.h/.cpp`

**Задачи:**
- [ ] `V8File`: абстракция для работы с файлами .cf/.1CD
- [ ] `V8Catalog`: каталог объектов конфигурации
- [ ] Интеграция с `V8Container` для навигации по структуре
- [ ] Поддержка версий формата 15 и 16

**Критерии готовности:**
- ✅ Чтение файлов .cf/.1CD без ошибок
- ✅ Построение дерева объектов конфигурации

#### 1.4. Управление памятью
**Задачи:**
- [ ] Аудит всех классов на использование сырых указателей
- [ ] Замена на `std::unique_ptr` и `std::shared_ptr`
- [ ] Проверка на утечки памяти (Valgrind/AddressSanitizer)

## 📋 Технические требования

### 2.1. Стандарты кода
- C++17 (обязательно)
- Qt6 ≥ 6.7.0
- Использование умных указателей (`unique_ptr`, `shared_ptr`)
- RAII для управления ресурсами
- Const-correctness
- Исключения только для критических ошибок

### 2.2. Тестирование
- Фреймворк: GoogleTest / Catch2
- Покрытие: ≥80% для core-модулей
- Обязательные тесты:
  - Парсинг .1CD разных версий
  - Чтение всех типов объектов метаданных
  - Граничные случаи (пустые файлы, поврежденные данные)

### 2.3. Документация
- Doxygen-комментарии для всех публичных методов
- Примеры использования для каждого класса
- README с инструкциями по сборке

### 2.4. CI/CD
- GitHub Actions для Linux/Windows/macOS
- Автоматический прогон тестов при PR
- Сборка артефактов
