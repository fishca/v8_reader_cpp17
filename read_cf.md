# Алгоритм открытия файла CF (v8_reader)

## 1. Определение формата файла

Проверяется версия формата файла конфигурации 1С:
- **Format15**: 32-битные адреса, используется до версии 8.3.15
  - Размер заголовка: 16 байт
  - Смещение данных: стандартное
- **Format16**: 64-битные адреса, используется с версии 8.3.16
  - Размер заголовка: 20 байт
  - Смещение данных: 0x1359

## 2. Чтение заголовка файла

1. Файл открывается в бинарном режиме (`std::ios::binary`)
2. Считывается заголовок файла:
   - Для Format15: структура `stFileHeader` (32-bit)
   - Для Format16: структура `stFileHeader64` (64-bit)
3. Чтение выполняется по смещению `BASE_OFFSET`

## 3. Чтение таблицы размещения (TOC - Table of Contents)

1. Из заголовка файла извлекается адрес TOC
2. Считывается заголовок блока TOC (`stBlockHeader`)
3. Читается сама таблица размещения, содержащая:
   - Количество элементов
   - Массив структур `stElemAddr` для каждого элемента:
     - `header_addr` - адрес заголовка элемента
     - `data_addr` - адрес данных элемента
     - `name_hash` - хеш имени элемента

## 4. Цикл обработки элементов конфигурации

Для каждого элемента из TOC выполняется:

1. **Чтение заголовка элемента**:
   - Переход по адресу `header_addr`
   - Чтение имени элемента (UTF-16LE)
   - Чтение временных меток (creation time, modification time)

2. **Извлечение имени**:
   - Имя хранится в формате UTF-16LE
   - Преобразуется в UTF-8 для работы в кроссплатформенном коде

3. **Проверка наличия данных**:
   - Если `data_addr != UNDEFINED_VALUE`, читаются данные элемента

## 5. Чтение блоков данных

Данные могут быть разбиты на несколько страниц:

1. **Начало чтения**:
   - Переход по адресу `data_addr`
   - Чтение заголовка первого блока (`stBlockHeader`)

2. **Структура заголовка блока**:
   ```cpp
   struct stBlockHeader {
       uint32_t/uint64_t data_size;      // Размер полезных данных
       uint32_t/uint64_t page_size;      // Размер страницы
       uint32_t/uint64_t next_page_addr; // Адрес следующей страницы
       uint32_t flags;                   // Флаги (сжатие и др.)
   };
   ```

3. **Цикл чтения страниц**:
   - Чтение данных страницы
   - Если `next_page_addr != UNDEFINED_VALUE`:
     - Переход по адресу следующей страницы
     - Чтение следующего заголовка блока
     - Повторение цикла
   - Сборка всех страниц в единый буфер

## 6. Распаковка данных (zlib)

1. **Определение сжатия**:
   - Проверяются флаги в заголовке блока
   - Если данные сжаты, применяется распаковка

2. **Распаковка zlib**:
   - Используется raw deflate режим (`-MAX_WBITS`)
   - Уровень компрессии: `Z_BEST_COMPRESSION`
   - Функция `try_inflate()` для распаковки в память

3. **Стратегии распаковки**:
   - **SmartUnpack**: Выбор между распаковкой в память или во временный файл
   - Для больших данных предпочтительна распаковка в файл

## 7. Рекурсивная структура

1. **Вложенные CF-файлы**:
   - Некоторые элементы содержат вложенные файлы конфигурации
   - Например: общие модули, формы, макеты

2. **Рекурсивная распаковка**:
   - При обнаружении вложенного CF вызывается `Parse()` рекурсивно
   - Построение иерархической структуры элементов

## 8. Ключевые классы v8_reader

| Класс | Назначение |
|-------|------------|
| `CV8File` | Контейнер уровня файла, управление всем процессом распаковки |
| `CV8Elem` | Представление одного элемента конфигурации |
| `Format15<T>` | Шаблон для работы с 32-битным форматом |
| `Format16<T>` | Шаблон для работы с 64-битным форматом |
| `stBlockHeader` | Заголовок блока данных |
| `stElemAddr` | Элемент таблицы размещения |

## 9. Точки входа API

### Основные методы CV8File:

1. **`Parse(const std::string& filename)`**:
   - Основная функция распаковки файла
   - Возвращает список элементов верхнего уровня

2. **`ListFiles()`**:
   - Получение плоского списка всех элементов
   - Используется для индексации и поиска

3. **`LoadFileFromFolder(const std::string& path)`**:
   - Упаковка распакованной конфигурации обратно в CF
   - Рекурсивный обход директории

### Методы CV8Elem:

1. **`GetData()`**:
   - Получение сырых данных элемента
   - Автоматическая распаковка при необходимости

2. **`GetSubElements()`**:
   - Получение вложенных элементов
   - Для доступа к иерархии

3. **`SaveToFile(const std::string& path)`**:
   - Сохранение элемента в файл

## 10. Обработка ошибок

1. **Проверка целостности**:
   - Валидация заголовков
   - Проверка контрольных сумм (если присутствуют)

2. **Исключения**:
   - `std::runtime_error` при чтении несуществующих адресов
   - `std::invalid_argument` при неверном формате файла

3. **Логирование**:
   - Отладочная информация о процессе распаковки
   - Ошибки чтения блоков

## 11. Оптимизации

1. **Ленивая загрузка**:
   - Данные элементов загружаются только при запросе
   - Экономия памяти для больших конфигураций

2. **Кэширование**:
   - Кэш распакованных данных
   - Избегание повторной распаковки

3. **Потоковая обработка**:
   - Возможность обработки больших файлов без полной загрузки в память

## 12. Пример использования

```cpp
CV8File v8File;
try {
    // Распаковка файла
    auto elements = v8File.Parse("1Cv8.cf");
    
    // Обход элементов
    for (const auto& elem : elements) {
        std::cout << "Element: " << elem.GetName() << std::endl;
        
        // Получение данных
        auto data = elem.GetData();
        
        // Обработка вложенных элементов
        auto subElements = elem.GetSubElements();
    }
} catch (const std::exception& e) {
    std::cerr << "Error: " << e.what() << std::endl;
}
```

## 13. Особенности реализации

- **Шаблонный подход**: Единый код для 32- и 64-битных форматов через шаблоны
- **RAII**: Автоматическое управление ресурсами (файлы, память)
- **Кроссплатформенность**: Использование стандартных типов фиксированной ширины
- **Поддержка Unicode**: Корректная работа с именами на русском языке
